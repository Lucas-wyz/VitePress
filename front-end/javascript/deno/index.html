Deno #

Deno手册 GitHub #

A secure runtime for JavaScript and TypeScript.

一个安全的JavaScript和TypeScript的运行时。

特点 #

  • 尽可能与 Web 兼容，例如通过使用 ES 模块，并支持fetch。

  • 默认安全。除非明确启用，否则不能访问文件、网络或环境。

  • 开箱即用地支持 TypeScript。

  • 只提供一个可执行文件。

  • 具有内置实用程序，例如依赖项检查器 ( deno info) 和代码格式化程序 ( deno fmt)。

  • 有一组经过审查（审计）的标准模块，保证可以与Deno一起使用：deno.land/std

  • 可以将脚本捆绑到单个 JavaScript 文件中。

安装 #

Deno 是单个二进制可执行文件，没有依赖项， 适用于 macOS、Linux 和 Windows。

从发布页面下载发布二进制文件。 #

在 Linux 和 Windows 上，仅支持 x64。

在 macOS 上，提供了 M1 (arm64) 和 Intel (x64) 可执行文件。

可以使用下面的安装程序安装 #

Shell (Mac, Linux):

curl -fsSL https://deno.land/x/install/install.sh | sh

PowerShell (Windows):

curl -fsSL https://deno.land/x/install/install.sh | sh

更多请查看官网。

测试您的安装 #

运行deno --version.如果这将 Deno 版本打印到控制台，则安装成功。

更新 #

deno upgrade

安装特定版本的 Deno

deno upgrade --version 1.0.1

设置编辑器 #

以Visual Studio Code为例 #

安装官方扩展Deno 。安装后，它将连接到 Deno CLI 中内置的语言服务器。

安装时请阅读扩展的说明文档

由于大多数人在混合环境中工作，因此扩展不会像默认启用 Deno 那样启用工作区，并且需要 "deno.enabled"设置标志。您可以自己更改设置，也可以Deno: Initialize Workspace Configuration从命令面板中进行选择以启用您的项目。

权限 #

Deno 默认是安全的。除非您专门启用它，否则使用 Deno 运行的程序没有文件、网络或环境访问权限。访问安全敏感功能要求已通过命令行标志或运行时权限提示向正在执行的脚本授予权限。

权限列表 #

以下权限可用：

  • **--allow-env=<allow-env>**允许环境访问以获取和设置环境变量。从 Deno 1.9 开始，您可以指定一个可选的、以逗号分隔的环境变量列表，以提供允许的环境变量的允许列表。

  • --allow-hrtime允许高分辨率时间测量。高分辨率时间可用于计时攻击和指纹识别。

  • **--allow-net=<allow-net>**允许网络访问。您可以指定一个可选的、以逗号分隔的 IP 地址或主机名列表（可选带有端口）以提供允许的网络地址的允许列表。

  • --allow-plugin允许加载本机插件。请注意，插件不在沙箱中运行，因此没有与 Deno 进程相同的安全限制。因此，请谨慎使用。请注意 --allow-plugin 是一个不稳定的特性。

  • **--allow-read=<allow-read>**允许文件系统读取访问。您可以指定一个可选的、以逗号分隔的目录或文件列表，以提供允许的文件系统访问权限列表。

  • **--allow-run=<allow-run>**允许运行子进程。从 Deno 1.9 开始，您可以指定一个选项，以逗号分隔的子进程列表，以提供允许的子进程的允许列表。请注意，子进程不在沙箱中运行，因此没有与 deno 进程相同的安全限制。因此，请谨慎使用。

  • **--allow-write=<allow-write>**允许文件系统写访问。您可以指定一个可选的、以逗号分隔的目录或文件列表，以提供允许的文件系统访问权限列表。

  • -A, --allow-all允许所有权限。这将禁用所有安全性。

详情请查看 手册-权限
